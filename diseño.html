Diseño de Software

Principios, patrones y buenas prácticas para crear software mantenible y eficiente

¿Qué es el Diseño de Software?

Actividad en la que se plantea cómo resolver un problema a través de software. Incluye decisiones sobre:

  • Almacenamiento de datos
  • Interfaces de usuario
  • Comunicación con otros servicios
  • Delimitación de responsabilidades
Patrones de Diseño

Soluciones reutilizables a problemas comunes que proporcionan:

  • Lenguaje compartido
  • Estructura probada
  • Código más eficiente y mantenible
  • Guías para sistemas robustos y flexibles
Principios SOLID
Responsabilidad Única (SRP)

Una clase debe tener una sola razón para cambiar.

Abierto-Cerrado (OCP)

Abierta a extensión, cerrada a modificación.

Sustitución de Liskov (LSP)

Subclases sustituibles por sus clases base.

Segregación de Interfaces (ISP)

Interfaces específicas mejor que una general.

Inversión de Dependencia (DIP)

Depender de abstracciones, no de implementaciones.

Buenas Prácticas
Conocer Antipatrones

Evitar malas prácticas como clases con demasiado código o demasiadas clases con pocas responsabilidades.

Documentación

Fundamental para futuras actualizaciones, mantenimiento e incorporación de nuevos desarrolladores.

Selección Arquitectónica

Aplicar estilos y patrones apropiados para el tipo y tamaño del sistema.

GRASP

General Responsibility Assignment Software Patterns

Patrones generales para asignación de responsabilidades en el diseño orientado a objetos.

Complementa los principios SOLID para un diseño más robusto.
